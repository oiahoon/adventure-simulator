# 人格画像系统开发计划（行为证据版）

更新时间：2026-03-02  
适用范围：`app/main.js`、`ui/game-ui.js`、`content/*`、`docs/product/*`

## 目标
- 在“都市生存 100 天”框架下，新增可解释的人格画像总结。
- 画像基于整局行为证据与因果链，不做“单题即定型”。
- 输出必须包含：画像结论、置信度、关键证据、下一局建议。

## 设计原则
- 证据优先：只用可追溯的行为数据（选项标签、属性增减、连锁后果、里程碑决策）。
- 低偏差：减少一次极端选择对最终结果的主导权重。
- 可解释：每个画像都要能回溯到 Day + 事件 + 选择。
- 非诊断：明确声明“行为画像不等同临床心理评估”。

## 参考框架（题库映射基线）
- Big Five / IPIP 公开量表：用于构建“维度化而非二元化”的行为评分思路。  
  参考：IPIP-NEO 公开题项索引 [ipip.ori.org](https://ipip.ori.org/newNEOKey.htm)
- 小样本高效率量表设计经验：MINI-IPIP（20题）强调维度覆盖与短量表稳定性。  
  参考：Donnellan et al., 2006 [PubMed](https://pubmed.ncbi.nlm.nih.gov/16393070/)
- 情景判断测试（SJT）方法：用“情景-选项”而非直接自陈题，贴合游戏事件结构。  
  参考：SJT 综述与应用 [Wikipedia](https://en.wikipedia.org/wiki/Situational_judgement_test)

## 与游戏系统的映射
- 输入信号：
  - 事件标签：`risk/control/rest/work/network/social/content/money`
  - 决策后属性变化：`money/energy/mood/reputation/heat`
  - 因果旗标：`debt_spiral/budget_mode/overwork_line/...`
  - 连锁后果与里程碑记录：来自 `history`
- 输出维度（v1）：
  - 风险校准（稳健控盘 vs 高波动冲锋）
  - 社交驱动（独立推进 vs 关系协同）
  - 压力恢复（透支硬顶 vs 弹性恢复）
  - 规划视角（短线止血 vs 长线经营）

## 分阶段计划
1. P1（已开始）行为画像核心引擎
   - 建立维度权重表：标签权重、属性权重、旗标权重。
   - 计算标准化分值与置信度。
   - 输出 Top 证据链与画像建议。

2. P2 事件库扩容与题库化标签
   - 为每个事件补充“维度标签贡献”元数据（避免只依赖 tag 粗粒度）。
   - 引入“镜像题”设计：同一压力情景下不同策略，以减少措辞偏差。
   - 每个维度目标新增 20-30 条有效事件信号。

3. P3 校准与偏差控制
   - 固定 seed 批量仿真，检查分布是否过度偏单侧。
   - 角色偏差检查：不同开局角色同策略是否得分接近。
   - 置信度阈值策略：低证据时只给“倾向观察”，不下强结论。

4. P4 结局体验与分享优化
   - 结局页显示“画像 + 证据 + 置信度说明”。
   - 分享文案加入画像标题与 1 条高置信证据。
   - DeepSeek 故事总结与画像结论做一致性提示，避免冲突表达。

## 验收标准
- 画像结果稳定：同 seed 多次运行结论一致。
- 画像可解释：至少 2-3 条可追溯证据。
- 低样本保护：短局或异常局不会给高置信强结论。
- 与主玩法不冲突：结局仍以“生存复盘”为主，画像为增强层。
