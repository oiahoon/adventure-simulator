# 微信传播导向重设计（V3）

## 核心目标

- 从“中长局战斗”转为“3~5 分钟短局选择游戏”。
- 以手机单手操作为主，适配微信内浏览器。
- 每局结束可生成挑战文案 + 挑战码，支持群传播复刻。

## 新玩法定义

- 名称：城市生存7天。
- 核心循环：`事件 -> 三选一 -> 属性变化 -> 下一天`。
- 局长：7 天固定长度。
- 核心属性：现金、体力、心态、人设、热度。
- 失败条件：现金/体力/心态/人设任一归零。
- 结算：根据综合分与是否存活给出结局。

## 操作与人性化

- 所有关键交互均为大按钮，避免误触。
- 每天只做一次主决策，降低认知负担。
- 每局提供 2 个一次性“救命技能”（打盹、借钱）增强可控性。
- 实时展示每个选项的影响摘要，减少盲选挫败感。

## 微信传播机制

- 局末生成 `挑战码`（由 seed + 结局摘要编码）。
- 一键复制分享文案：分数、结局、挑战码。
- 支持输入挑战码复刻同一局事件序列。

## 内容策略

- 事件语境聚焦：租房、加班、副业、群聊舆论、人情压力。
- 每个事件提供 3 个风格差异明显的选项（稳健/社交/冒险）。
- 保留贴吧语感，但使用去标识化改写，不直接复用原帖。

## 技术落地

- 前端单页：`app/main.js + ui/game-ui.js + public/index.html`。
- 运行时文本状态：继续暴露 `window.render_game_to_text`。
- 自动化测试：保持 Playwright 冒烟与基础 Node 单测。

## 下一阶段（V3.1）

- 增加“局内随机连锁后果”（前一日决策影响下一日选项）。
- 增加微信群投票式玩法（双人对抗挑战码）。
- 增加分享图卡（canvas 导出）而非纯文本。
