# 剧情结构梳理与 us-killline 差异对比（2026-03-02）

## 1. 适用范围与对比前提

1. 本文基于当前仓库实现与文档：`app/main.js`、`content/story-events.js`、`docs/product/*.md`。
2. 仓库内未包含 `us-killline` 的源码/设计文档。
3. 因此“差异对比”采用产品层对比法：将 `us-killline` 视为“阈值驱动、斩杀线导向、线性生存压力玩法”的代表范式，不对其未公开实现细节做断言。

## 2. 当前游戏设计总览（我方）

1. 目标节奏：文案目标“坚持 100 天”，实际可持续游玩直到关键属性归零。
2. 核心资源：现金、体力、心态、人设、热度（热度放大风险但非直接败因）。
3. 叙事驱动：章节式因果链 + 状态插曲 + 强制生存事件 + 成长事件。
4. 调度优先级：开局事件 > 强制事件 > 成长事件 > 状态插曲 > 章节主线事件。
5. 难度来源：即时收益与延迟后果叠加，且债务/过劳/舆论三压力量表会触发系统性损耗。

## 3. 主线剧情骨架（当前实现）

1. 第一章 开局压力：房租、绩效、内容爆火、人情求助、社群风口。
2. 第二章 路线放大：按第一章选择分流到“债务扩散”或“工位风暴”。
3. 第三章 舆论/喘息分岔：高风险路线进入舆论反噬，稳健路线进入短暂喘息。
4. 第四章 身体与关系：过劳线触发健康危机，稳态线进入关系拉扯。
5. 第五章 热度或现金：高热度进入流量博弈，中低热度进入现金保命。
6. 第六章 摊牌或转型：债务摊牌线与转型经营线收束，决定后续长期生存姿态。

## 4. 支线体系（当前实现）

1. 强制支线（硬门槛）：低体力生病、重度透支、断粮预警，会中断日常并强制处理。
2. 状态插曲（软触发）：低现金/低心态/低体力触发临时事件，补偿与代价并存。
3. 成长支线（里程碑解锁）：债务生存术、边界感、舆论打法、稳定后升级。
4. 延迟后果链：高风险/高负荷/高曝光操作会在 1~2 天后回收代价或收益。
5. 退场机制：`noRepeat/maxRepeats/retireWhen` 控制事件重复与章节退场，避免无限刷同题材事件。

## 5. 可能扩展的主线与支线（建议入库）

### 5.1 主线扩展候选

| 主线名 | 触发前提 | 关键抉择 | 典型后果 |
|---|---|---|---|
| 规则突变线 | 热度>=6 或 内容线累计 | 跟规则/钻规则/退场观察 | 收益波动增大、稳定性下降 |
| 家庭责任线 | Day>=20 且 现金或心态偏低 | 扛责任/谈边界/资源互换 | 人设与现金强耦合 |
| 城市迁移线 | 房租压力持续 + 稳定里程碑未达成 | 留守/换城/短租过渡 | 成本结构与社交网络重算 |
| 机会窗口线 | 稳定里程碑达成 | 升级技能/创业试水/继续保守 | 中长期成长性分叉明显 |

### 5.2 支线扩展候选

| 支线名 | 触发前提 | 玩法作用 |
|---|---|---|
| 身体修复周期线 | 连续低体力 + 强制事件次数>=2 | 引入恢复期，不再只靠一次“急救” |
| 信任修复线 | 人设<4 且关系线活跃 | 补充“信誉可恢复”而非单向崩坏 |
| 平台黑盒线 | 热度高且内容线重复 | 增加不确定反馈，抑制套路解 |
| 小确幸缓冲线 | 心态低但现金/体力尚可 | 提供低收益高稳定的情绪修复选项 |

## 6. 与 us-killline 的核心差异（产品层）

| 维度 | 当前项目（都市生存100天） | us-killline（范式假设） |
|---|---|---|
| 目标表达 | 100天生存叙事 + 可继续冲记录 | “斩杀线/阈值”导向的短周期压测 |
| 叙事结构 | 章节主线 + 多支线 + 延迟后果 | 线性压力堆叠，叙事弱于数值反馈 |
| 事件来源 | 贴吧/微博热梗 + 都市现实事件语境 | 通用生存或阈值压力题材为主 |
| 分支深度 | 同章多分支、跨章因果回收、事件退场 | 常见为局内分叉，跨章回收较弱 |
| 难度形态 | 即时效果 + 压力系统 + 延迟回收 | 直接扣减与阈值判定占主导 |
| 失败解释 | 结局成因 + Top3因果节点 + Top3决策 + 整局故事 | 常见为“达到/跌破阈值”的结果说明 |
| 分享传播 | 微信导向文案 + 剧情复盘 + 角色语境 | 常见为分数/天数/阈值成绩分享 |

## 7. 后续文档与实现对齐建议

1. 先补“主线-支线映射表”的机器可读版本（JSON 或 TS 常量）并接入事件调度器。
2. 在结算复盘中增加“本局走到的主线ID + 支线命中列表”，让剧情解释可验证。
3. 扩展事件池时强制附带“前置条件、后续回收、退场条件”三元字段，避免再回到随机梗堆积。
4. 若后续拿到 `us-killline` 实际文档/代码，再将第 6 节由“范式对比”升级为“实现级对比”。
