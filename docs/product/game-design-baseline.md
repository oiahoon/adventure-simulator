# 游戏设计基线（V4）

## 产品定位

- 题材：面向中国玩家的都市生存卡牌选择 Web Game。
- 平台：手机浏览器优先（微信内打开可玩），桌面兼容。
- 单局体验：轻量、可复盘、可分享。

## 核心循环

1. 每天触发一个因果事件。
2. 玩家从三个选项中决策。
3. 状态即时变化，且可能埋下延迟后果。
4. 进入下一天，直到触发失败结局。

## 当前玩法约束

- 目标文案：挑战坚持 100 天（不主动提示“通关后可继续”）。
- 失败条件：`现金/体力/心态/人设` 任一归零。
- 状态展示：现金、体力、心态、人设、热度（0~10）。
- 每天可使用 1 次临时技能，技能每日刷新。

## 因果与难度策略

- 明线因果：章节事件由 `flags + 当前状态` 驱动，不是纯随机。
- 暗线压力：债务/过劳/舆论压力累计后触发系统性损耗。
- 延迟后果：部分高风险决策在 1~2 天后回收代价。
- 反套路：同类策略重复使用会提高翻车概率、降低稳定收益。

## 分享与传播

- 结局提供一键复制分享文案。
- 分享内容包含：生存天数、分数、结局、可点击游戏链接。
- 不再使用挑战码复刻机制。

## 结局体验

- 结局页输出：结局标题、成因摘要、最近决策链。
- 调用 DeepSeek 生成“整局故事”复盘。
- 复盘文案使用生活化表达，避免生硬属性术语。
