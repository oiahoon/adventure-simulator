# 事件因果链设计（核心内容）

## 设计目标

- 让玩家感受到“我当前处境导致了下一步事件”，而不是随机抽卡拼接。
- 保持单局可复盘：能清晰回答“我为什么赢/输”。

## 因果链模型

采用三层触发结构：

1. `Queue`（强制后续）
2. `Arc`（叙事弧阶段）
3. `Deck`（通用机会池）

优先级：`Queue > Arc > Deck`

## 核心数据字段

- `eventId`: 事件唯一标识
- `source`: `queue | arc | deck`
- `when`: 触发条件
- `effects`: 状态变化
- `enqueue`: 后续强制事件
- `flags`: 因果标记（可带 TTL）
- `bias`: 对后续事件权重的影响

## 因果链模板

### 模板 A：压力升级链（短链）

1. 触发事件：`收入下降`
2. 状态变化：`cash -x, stress +y`
3. 入队：`房东催租`（1~2 回合内必须出现）
4. 分支结果：
   - 顶住压力：现金降更多，信誉保留
   - 借贷周转：债务上升，短期生存提升

### 模板 B：机会反噬链（中链）

1. 触发事件：`高回报兼职机会`
2. 结果 A：收入增长 + 疲劳积累
3. 结果 B：保守拒绝 + 安全但发展慢
4. 后续回收：
   - 若疲劳过高，触发`身体透支`支线
   - 若连续拒绝，触发`机会流失`支线

## 当前版本应落地的 5 条主因果弧

1. 生存成本弧：收入 -> 租房 -> 借贷 -> 风险
2. 职业压力弧：绩效 -> 裁员风声 -> 转型/硬扛
3. 健康消耗弧：加班 -> 疲劳 -> 崩溃/修复
4. 社会关系弧：人情互助 -> 信任/反噬
5. 舆情热度弧：短期关注 -> 机会/副作用

## 复盘输出要求

每局至少输出：

- `Top 3` 关键因果节点
- `Top 3` 导致失败或成功的决策
- 一条可读摘要：`事件A -> 决策B -> 结果C`

## 设计红线

- 不能只堆“梗事件”而没有状态后果。
- 不能出现“关键事件触发后无后续回收”。
- 不能让强制后续被随机池长期淹没。
