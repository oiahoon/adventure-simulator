# 开发执行路线图（M0-M4）

## M0 项目骨架（已完成）
- 目标：建立新项目目录、可运行页面、日志基础、自动化可读状态。
- 任务：
  - 初始化 `core/content/ui/app/public/tests`。
  - 提供 `window.render_game_to_text` 与 `window.advanceTime`。
  - 建立最小单测链路（`npm test`）。
- 验收：
  - 页面可打开并渲染基础框架。
  - 测试通过。

## M1 单场战斗 MVP（已完成）
- 目标：完成一场可胜负判定的回合制战斗。
- 任务：
  - 战斗状态机与回合循环。
  - 至少 12 张基础卡（攻/防/功能）。
  - 3 个敌人模板。
  - 基础战斗 UI（手牌、能量、结束回合）。
- 验收：
  - 可完整打一局并出现胜/负结果。

## M2 玩法强化（已完成）
- 目标：增强策略深度与反馈可读性。
- 任务：
  - 状态系统（中毒、脆弱、力量等）。
  - 条件化 AI 决策。
  - 战斗日志与关键反馈。
- 验收：
  - 存在可验证的状态组合打法。

## M3 局外成长（已完成）
- 目标：构建单局流程与成长体验。
- 任务：
  - 战后三选一奖励。
  - 加牌/删牌简易构筑。
  - 关卡节点（普通战、精英战）。
- 验收：
  - 单局存在多个节点推进与构筑决策。

## M4 打磨与发布（已完成）
- 目标：移动端可用、可交付测试版。
- 任务：
  - 新手引导与规则提示。
  - 数值首轮平衡。
  - 交互和性能优化。
  - 发布检查清单。
- 验收：
  - 完整跑通一局且移动端体验稳定。
