# 全新卡牌 Web Game 重构计划（执行中）

## 2026-02-27 Reboot 决策

用户要求放弃既有 UIUX 与玩法呈现，转为真正“从零开始”的新卡牌 Web 游戏。
本计划切换为 v4 重启路线：先做独立前端引擎和全新视觉壳，再逐步迁移事件深度与协议统一。

## 目标

在保留当前引擎能力的前提下，重建一套更现代、移动优先、策略反馈更明确的卡牌 Web UI。

## 阶段

### Phase A：入口与兼容策略
- `/game/` 切换到新重构版。
- 旧壳保留到 `/game-classic/`。
- legacy 壳继续保留到 `/game-legacy/`。

### Phase B：核心交互重建
- 主卡滑动 + 按钮兜底。
- 机会手牌管理（出牌/弃置/延后）。
- 机会偏好切换（balanced/survival/growth/debt）。

### Phase C：视觉与动效增强
- 新视觉风格（层次背景、信息分区、移动端优先）。
- 入场动效、半滑动反馈、KPI 变化反馈。

### Phase D：复盘与分享升级
- 关键牌 + 高影响牌 + 亮点摘要。
- 分享图与分享文案同步复盘结构。

### Phase E：验证与收口
- 回放、schema、事件检查通过。
- 文档与进度同步。

### Phase F（v4）：独立前端引擎重启
- 不依赖旧版 v2 API 的最小可玩内核（new/draw/play/end）。
- 新卡池 + 条件触发 + 事件链因果 + 成就与分享文案。
- 输出 `render_game_to_text` 与 `advanceTime` 保持可测性。

- Phase E：进行中（自动化浏览器在当前沙箱受限）
