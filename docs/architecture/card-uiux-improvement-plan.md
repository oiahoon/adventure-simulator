# Card UI/UX 改进计划（按 cards-play-deep-research-report）

## 目标

将当前可玩壳层升级为“移动端单卡决策优先、桌面端手牌可经营、结算可复盘可分享”的卡牌体验。

## 分阶段计划

### Phase 0：计划与验收

- 固化 UI/UX 验收项：
  - 移动端主交互为单卡滑动二选一（支持按钮兜底）。
  - 卡面包含标题、正文、左右选项文案与影响预告。
  - 强制后续事件（queue）有显性提示与视觉区分。
  - 结局卡可复盘关键剧情链与关键决策。
  - 关键 UX 指标可导出用于 A/B。

### Phase 1：移动端单卡滑动

- `/game` 默认提供单卡区（mobile-first）。
- 拖拽距离达到阈值触发左/右出牌，未达阈值回弹。
- 半滑动阶段高亮左右影响预告。

验收：
- 手机竖屏可一手完成主要操作。
- 滑动与按钮都可完成同样决策。

### Phase 2：卡面语义与强制牌提示

- 补齐卡面信息字段：
  - `text`
  - `leftLabel`/`rightLabel`
  - `forced`
- 引擎发牌元数据输出这些字段，UI 直接渲染。
- 强制牌展示红色角标与队列“下一张强制牌”提示。

验收：
- 玩家能理解“这张卡是什么处境、左右各是什么决策”。
- 队列连锁可视化，不再像随机跳转。

### Phase 3：结局复盘卡增强

- 结局输出增加：
  - 关键牌 TopN
  - 关键剧情链结果
  - 最近关键轨迹摘要
- 分享图与复制文案对齐复盘结构。

验收：
- 结局页可直接截图传播并复述“我是怎么输/赢的”。

### Phase 4：最小埋点与 A/B 字段

- 记录：
  - 出牌行为（左/右）
  - 滑动决策占比
  - 强制牌命中
  - 分享按钮转化
- 在 `render_game_to_text` 增加 `ux_metrics` 字段。

验收：
- 可以比较不同交互方案下的深度/链路/分享指标。
