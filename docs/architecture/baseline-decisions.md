# 基线决议（单一真相）

生效时间：2026-02-28  
适用范围：`docs/*`、`app/*`、`ui/*`、`public/*`、`api/story/*`

## D1：局目标统一为 100 天

- 决议：玩法目标统一为“挑战坚持 100 天”。
- 约束：文档与 UI 不再使用“7天固定局长”表述。
- 备注：不在文案中强调“100天后可无限继续”，只呈现 100 天挑战目标。

## D2：移除挑战码复刻机制

- 决议：不再提供 `seed/challenge code` 导入与复刻。
- 保留：局末一键复制分享文案 + 可点击游戏链接。
- 约束：文档与页面中不得出现“挑战码、复刻这局、导入挑战码”等入口描述。

## D3：Core Action 架构定位为兼容层

- 决议：当前线上主玩法以前端事件模式为主路径（`app/main.js + ui/game-ui.js + public/index.html`）。
- 定位：`core/* + api/mud/* + bin/mud-cli.js` 作为 Legacy 兼容与回归资产保留。
- 约束：涉及线上主玩法的需求，默认在主路径实现并同步文档；
  若改为 Core 主路径，必须先更新本决议并给出迁移计划。

## 冲突处理规则

1. 若任一文档与本决议冲突，以本文件为准。
2. 提交涉及玩法目标/传播机制/主架构变更时，必须同步更新本文件。
