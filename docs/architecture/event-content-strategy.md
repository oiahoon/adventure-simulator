# 事件与内容策略

## 题材方向

- 都市生存压力：收入、风险、状态博弈
- 语言风格：中文互联网语境，但避免真实主体映射

关键设计依据：

- `docs/product/event-causal-chain-design.md`
- `docs/product/tieba-content-integration.md`

## 内容扩展原则

- 小步迭代：先扩卡牌与敌人，再扩复杂事件链
- 数据驱动：新内容优先配置化，不写死在 UI
- 可测试：新增内容必须可被回归测试覆盖
- 可回收：所有高影响事件必须有后续因果回收节点

## 后续目标

- 引入可回放事件日志
- 建立内容 schema 校验与 lint
- 建立事件包发布流程（按版本或季度）
- 建立“贴吧素材 -> 事件原型 -> 风险审查 -> 灰度上线”的流水线
