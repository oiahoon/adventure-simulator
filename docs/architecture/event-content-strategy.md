# 事件与内容策略（当前）

## 题材方向

- 都市生存压力：收入、过劳、关系、舆论、心理波动。
- 语言风格：中文互联网语境，贴吧老哥式调侃，但不过线。

## 当前内容结构

- 主事件池：内联于 `app/main.js`，按章节和因果条件触发。
- 素材种子：`content/tieba-event-seeds.js`（用于后续扩容改编）。
- 叙事总结：`api/story/summary.js` 调用 DeepSeek 生成整局故事。

## 内容扩展原则

- 可因果回收：高影响事件必须有后续代价或补偿。
- 可复盘：玩家能从历史链路看懂“为什么输/赢”。
- 可控风险：去标识化改写，不复制真实主体表达。
- 可测试：新增事件不应破坏结算与失败判定。

## 后续目标

1. 将贴吧种子半自动映射到章节池，降低手工维护成本。
2. 增加事件冷却与主题轮换，降低中后期重复感。
3. 输出结构化复盘 Top3 因果节点，强化故事解释力。
