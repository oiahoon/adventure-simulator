# 测试与质量策略（当前）

## 测试分层

- 单元/行为测试：当前维护在 `tests/*`（历史运行时回归）。
- 前端冒烟：Playwright + `window.render_game_to_text`。
- 线上可用性核对：首屏加载、事件可选、结局可生成、分享可复制。

## 质量门禁

- 每次提交必须通过 `npm test`。
- 发布前至少执行一次移动端视口手工回归。
- 规则改动需同步 `docs/product/*` 与 `progress.md`。

## 当前风险点

- 文档与代码偏差（玩法目标/传播机制）会导致错误迭代。
- 样式变更可能破坏手机首屏可操作性。
- DeepSeek 未配置时，结局故事会降级为“暂未生成”。

## 建议补强

1. 增加一条移动端布局快照断言（首屏包含至少 1 个可点选项）。
2. 为结局复盘增加结构化断言（关键因果节点数量）。
3. 给分享文案增加内容格式校验测试。
