# 《朕的一生》游戏策划案 (Game Design Document) v0.1

**版本**: 0.1  
**日期**: 2026‑04‑19

## 1. 高概念

《朕的一生》是一款基于卡牌决策的皇帝模拟游戏，灵感来自《Reigns》【247434138540471†L18-L25】。玩家扮演虚构王朝的帝王，通过左右滑动卡牌做出决策，在“民心、国库、兵权、朝政”四项资源间保持微妙平衡【247434138540471†L50-L59】。任何资源过高或过低都可能导致叛乱、篡位、宗教裁判等结局【247434138540471†L55-L59】。游戏通过简短而幽默的事件和多样化死因构建跨世代叙事，让玩家的错误决定成为新的故事【247434138540471†L64-L67】。

## 2. 世界观与故事背景

游戏设定于一个融合汉唐宋等文化元素的虚构王朝。外有藩镇割据与北疆侵扰，内有皇亲国戚、宦官、士林和江湖门派；自然灾害与民间传说交织。玩家身处这盘棋局，既要维持国家稳定，又要应对个人欲望、皇位继承和超自然奇谈。

## 3. 平台与目标人群

- **平台**：微信 H5 和移动浏览器。  
- **目标人群**：喜欢轻策略和互动叙事的用户，对历史或武侠题材有兴趣的玩家。

## 4. 核心玩法与循环

### 4.1 资源系统

四项资源条分别为：

- **民心**：百姓支持度，过低会爆发起义，过高可能导致期望过高。
- **国库**：财政储备，耗尽时政府无法运作，过富可能引发贪腐。
- **兵权**：军队力量，过弱时边患或内乱爆发，过强则武将权力膨胀。
- **朝政**：权力平衡和朝廷稳定度，过低导致权臣篡权，过高则政治僵化。

这些指标的平衡类似《Reigns》四条势力（教会、人民、军队、金库）的机制【247434138540471†L50-L59】。

### 4.2 决策交互

每回合展示一张事件卡，含人物头像、事件描述和两种处理方式。玩家左右滑动选择，系统即时更新四维资源，并可能设置或清除标记、增加计数器等【247434138540471†L84-L89】。简洁的对话文案配合古风幽默让玩家迅速做出决策。

### 4.3 事件系统与概率算法

#### 4.3.1 事件结构

每张事件卡包含以下字段：

- `id`：唯一标识；
- `角色`：发起事件的角色或场景，如宰相、将军、宦官、方士、江湖侠客、灾民；
- `描述`：事件的文本内容；
- `左/右选项`：两种简短回应或行动；
- `资源影响`：对民心、国库、兵权、朝政的增减数值；
- `状态变更`：对标记(flag)或计数器(counter)的修改；
- `触发条件`：只有在特定状态满足时才能抽取，如战争期间、已立太子等；
- `权重`：随机抽取时的概率权重；
- `后续事件`：若选项引入特定事件链，则在后续回合中添加相应卡牌。

#### 4.3.2 概率袋算法

- **初始化**：游戏维护一只“概率袋”，包含所有事件卡。  
- **过滤阶段**：抽卡前先移除与当前王国状态不符的卡牌，例如没有皇后时移除与婚姻相关的事件【428463298753366†L263-L276】。  
- **权重分配**：为剩余卡牌赋予不同大小（权重）。某些剧情链的卡牌体积更大，从而提高被抽取概率【428463298753366†L278-L291】。  
- **抽取阶段**：从袋中随机选出一张卡作为当前事件【428463298753366†L283-L286】。  
- **动态调整**：事件结束后可能移除相关卡牌，或添加新的卡牌集合，使概率袋随事件发展而扩缩【428463298753366†L293-L302】。部分事件进入“小袋”子系统，只包含几张卡牌，用于战斗、地牢等特殊场景【428463298753366†L306-L309】。

这种设计保证既有随机性，又能依据玩家选择呈现逻辑连贯的故事【428463298753366†L349-L354】。

### 4.4 隐藏状态与因果链

- **标记 (flag)**：布尔或枚举变量，如 `war_ongoing`、`eunuch_power`、`famine_risk`、`taizi_established`。  
- **计数器 (counter)**：累加行为次数或强度，如 `tax_anger_level`、`eunuch_favor_points`。  
- **事件链**：由多张连续事件组成，如“宦官专权链”“藩镇叛乱链”“灾荒民变链”“方士求仙链”。链条包括起始事件、累积事件、危机事件和结局/收束。

即使只有少数事件依赖历史状态，该机制也让玩家觉得每张卡都可能与过去有关【428463298753366†L349-L354】。

### 4.5 死亡与结局

- **极值死亡**：任一资源条达到 0 或 100 会触发死亡。例如在《Reigns》中，当金库清空且军队过低或教会过高时，会导致国王溺亡【632964269596715†L168-L173】。本作将设计对应的中国式死法，如被叛军围攻、死于政治逼宫等。
- **特殊死亡**：某些死亡由事件链或标记决定，如放任宦官权力膨胀致宫廷政变，或长期宠信方士导致服丹暴毙。  
- **和平退位/成就结局**：若长期维持平衡并达成特定条件，可触发“明君善终”“遁隐江湖”等结局。  
- **称号系统**：根据玩家决策偏好、资源状态与标记赋予称号，如“仁德天子”“暴君”“傀儡皇帝”“九千岁之友”等，用于生成分享页。  

参考《Reigns》中 29 种死亡原因的设计，各种死亡结果会鼓励玩家探索不同道路【632964269596715†L168-L173】。

### 4.6 世代循环

一局结束后，玩家转生为下一任皇帝。部分世界状态（如战争状态、建筑、势力态度）可继承，而四维资源重置。跨世代任务（如修建长城、平定边乱）需要多位皇帝共同完成，增加长期目标感。

## 5. 角色与事件设计

### 5.1 主要角色原型

- **宰相**：上奏政策改革、税收、官员任命等，影响朝政与国库。
- **将军**：汇报边疆战况、军费和剿匪事宜，影响兵权与民心。
- **太后/外戚**：提出婚姻、礼仪和继承要求，牵动朝政与民心。
- **宦官**：掌握宫中权力，可引发权斗与政变。
- **文官/言官**：建议与弹劾，影响朝政稳定及民心。
- **方士/道士**：提供长生丹、炼丹或观星，触发超自然事件链。
- **江湖侠客**：引入武侠剧情，如门派纠纷、比武招亲等。
- **农民/商贾**：反映民生、税务和经济问题。

每个角色拥有多张基础事件卡，并至少贯穿一条剧情链。

### 5.2 事件链示例

1. **宦官专权链** – 扩招内廷→宦官势力上涨→言官弹劾→宫廷政变或“九千岁之友”结局。
2. **藩镇叛乱链** – 削藩失败→藩王积蓄实力→军阀割据→叛乱爆发，需要多轮镇压。
3. **灾荒民变链** – 天灾与连年加税→饥荒蔓延→赈济与否决定百姓态度→起义或安抚。
4. **太子立储链** – 是否立太子→皇子争权→废立与夺嫡→可能导致内乱或顺利继位。
5. **江湖盟主链** – 与门派合作或对立→江湖大会→比武决断→门派关系改变。

### 5.3 示例事件卡

```
id: "army_pay"
角色: 将军
描述: "边军久未得饷，士兵骚动，您要拨款吗？"
左选项: "拨款"
  资源影响: 国库 -15, 兵权 +10
  状态变更: war_payment_done = true
右选项: "暂缓"
  资源影响: 国库 +5, 兵权 -20
  状态变更: army_discontent += 1
触发条件: 无
权重: 5
```

```
id: "eunuch_audience"
角色: 宦官
描述: "司礼监请求增设中官名额，以便管理宫中大小事务。"
左选项: "批准"
  资源影响: 朝政 +5
  状态变更: eunuch_power += 1
右选项: "拒绝"
  资源影响: 朝政 -5
  状态变更: eunuch_resentment += 1

权重: 4
```

这些示例展示了如何通过简单文本表达事件，同时影响资源与隐藏状态。

## 6. 美术与音效

- **美术风格**：MVP 固定为 SBTI-lowpoly 风格：低饱和几何 cutout、可见多边形折面、人格测试头像感，并以小冠、官帽、袍形、奏折、剑、宫殿轮廓等轻量符号表达古代宫廷。完整规范见 `docs/art/sbti-lowpoly-style-guide.md`。
- **UI 布局**：顶部横条显示四维资源，并使用透明 HUD 图标；中部为事件卡，附角色头像；滑动时有卡牌拖拽和回弹动画；资源变化通过轻量数字、箭头和颜色反馈表现。
- **音效设计**：背景音乐以古筝、琵琶等民族乐器为主；操作反馈使用鼓点、锣声；死亡或成就事件有独特音效。

## 7. 技术架构

- **前端**：基于 HTML5、Canvas 或 WebGL 实现，支持触摸操作和滑动手势。
- **状态管理**：用 JavaScript/TypeScript 管理事件处理、概率袋逻辑和状态机。
- **数据驱动**：事件卡以 JSON/YAML 存储，便于策划编辑和版本控制。
- **存档系统**：利用本地存储保存跨世代状态和已解锁内容。
- **分享实现**：生成结果图（使用 Canvas 合成）并调用微信 JS-SDK 分享。

## 8. 测试与平衡

- **测试重点**：事件触发条件、概率袋算法、四维资源平衡、结局判定完整性。
- **BA 参与**：梳理事件逻辑表、状态机和标记变更矩阵，确保规则闭合。
- **QA 参与**：设计覆盖主要因果链的测试用例，验证极端情况，如四维极值和复杂事件组合；检查文本完整性。
- **平衡指标**：平均每局 5–10 分钟；不同策略有明显差异但不至于走向单一解；死亡原因分布均衡，鼓励探索。

## 9. 可扩展内容

- **特殊身份模式**：开局随机生成“寒门”“世家”或“江湖遗孤”等身份，改变初始资源和可用事件。
- **武侠篇章**：新增门派、秘籍和奇遇事件，引入比武、江湖任务等玩法。
- **多人模式**：后续可考虑让部分玩家扮演朝臣或门派代表，通过投票或阴谋影响皇帝决策。
- **成就与皮肤**：收集特定结局解锁特殊称号、头像或界面皮肤。

## 10. 开发计划概述

1. **原型阶段（4 周）**：实现四维系统、事件卡结构和概率袋算法；完成基础 UI。  
2. **内容搭建（6 周）**：策划撰写至少 100 张卡牌和 10 条事件链；美术绘制角色和界面；音效初步制作。  
3. **功能完善（3 周）**：完成分享系统、结果海报、成就系统，并加入存档功能。  
4. **调优与测试（4 周）**：通过内测和平衡测试调整数值与事件频率；修复 bug 并优化体验。  
5. **上线准备（2 周）**：整合微信分享和统计接口，制定上线宣传计划。

## 11. 结语

《朕的一生》通过极简的滑动交互与深度事件系统向《Reigns》致敬【247434138540471†L18-L25】，并通过概率袋算法和丰富的事件链提供变化多端的叙事体验【428463298753366†L263-L302】。结合中国历史与武侠文化，玩家将在欢笑与危机中体验皇权的沉浮。清晰的策划与 BA/QA 的系统化梳理可确保规则闭合、体验平衡，为后续扩展奠定坚实基础。
