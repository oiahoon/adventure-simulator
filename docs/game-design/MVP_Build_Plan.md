# 《朕的一生》MVP 构建拆分计划 v0.1

日期：2026-04-19

本计划把 `MVP_Implementation_Spec.md` 拆成可以逐步实现和验收的阶段。每个阶段都应该保持可运行，避免一次性堆完再调。

## M0：数据与静态界面骨架

目标：让项目能加载事件数据和资源清单，先看到静态游戏界面。

### 范围

- 建立前端项目骨架。
- 加载 `data/chinese-reigns/events.mvp.seed.json`。
- 加载 `data/chinese-reigns/objectives.mvp.seed.json`。
- 加载 `data/chinese-reigns/endings.mvp.seed.json`。
- 定义资源、角色、背景的路径映射。
- 展示一张静态事件卡。
- 展示顶部四资源 HUD。

### 验收

- 页面可在手机竖屏尺寸打开。
- HUD 显示民心、国库、兵权、朝政。
- HUD 使用 `icons/*-transparent.png`。
- HUD 不显示精确数值。
- 事件卡能显示 actor 头像、背景、文本、左右选项。
- 开局能显示 2 个本任目标。
- 控制台无资源 404。

### 暂不做

- 真正滑动。
- 资源变化。
- 抽卡。
- 结局。

## M1：核心决策循环

目标：玩家可以连续滑动卡牌，选择会改变资源。

### 范围

- 卡牌左右拖动。
- 拖动阈值和回弹。
- 提交选择。
- 应用 `resource`、`set_flag`、`counter` 三类效果。
- 显示资源变化反馈。
- 每次选择后切到下一张卡。
- 每处理一张卡，年份增加 1。

### 验收

- 连续处理 10 张卡不中断。
- 左右选择应用不同效果。
- 资源保持在 `0-100`。
- 资源危险区有明显提示。
- 选择预览只显示影响方向，不显示具体数值。
- 同一张卡不会立即重复出现。

## M2：概率袋与条件过滤

目标：事件出现开始有逻辑，而不是简单随机。

### 范围

- 支持 `conditions`。
- 支持 `weight`。
- 支持 `once`。
- 支持 `cooldown`。
- 避免同一 actor 连续出现超过 2 次。
- 针对危险资源提高补救事件权重。

### 验收

- `tutorial_first_memorial` 只出现一次。
- `censor_accuses_eunuch` 只在 `eunuch_power >= 1` 后出现。
- cooldown 内的卡不会被抽到。
- 资源危险时相关补救卡出现频率提高。

## M3：结局与结果页

目标：一局能结束，并生成有传播感的结果。

### 范围

- 极值结局判定。
- 高值结局判定。
- 特殊结局判定。
- 称号计算。
- 本任目标完成判定。
- 死亡时间线记录。
- 结果页展示。
- 本地记录已解锁结局。
- 再来一局。

### MVP 结局

- `rebellion`
- `coup`
- `empty_treasury`
- `military_takeover`
- `frontier_collapse`
- `peaceful_abdication`
- `old_age_succession`
- `expectation_revolt`
- `bureaucratic_suffocation`
- `corruption_flood`

### 验收

- 人为把任一资源推到 0 或 100 能进入对应结局。
- 连续长局不会超过 60 年，达到上限时进入寿终传位。
- 结果页包含称号、在位年数、死因/退位原因、点评。
- 结果页记录本任完成目标。
- 再来一局能重置资源和卡池。
- 已解锁结局保存在本地。
- 下一任皇帝开始时保留已解锁记录。

## M4：内容扩充到可试玩版本

目标：从 10 张 seed 卡扩到 40 张 MVP 卡，形成 5-10 分钟一局的节奏。

### 范围

- 基础朝政事件：8 张。
- 民生/灾荒事件：8 张。
- 国库/税务事件：6 张。
- 军事/边关事件：8 张。
- 宦官/后宫/继承事件：6 张。
- 稀有奇遇/方士事件：4 张。

### 验收

- 平均一局 5-10 分钟。
- 单局前 15 回合重复感不明显。
- 至少 5 个结局可自然触发。
- 至少 3 个称号有稳定触发路径。
- 没有明显“永远最优”的选择策略。

## M5：分享页与传播打磨

目标：把游戏从“可玩”推向“愿意晒”。

### 范围

- 使用 `share/result-card-bg.png` 合成分享图。
- 展示称号、年数、死因、资源倾向。
- 支持保存/长按图片。
- 预留微信分享接口。

### 验收

- 分享图中文字清晰。
- 分享图无资源遮挡。
- 结果文案有梗但不冗长。
- 不依赖服务端也能生成本地分享图。

## 风险与处理

| 风险 | 表现 | 处理 |
| --- | --- | --- |
| 事件太像选择题 | 每张卡都像加减分考试 | 文案必须有角色口吻和荒诞感。 |
| 资源变化太透明 | 玩家只追求数值最优 | preview 只显示方向，不显示具体数值，HUD 不显示数字。 |
| 死亡太突然 | 玩家不知道为何结束 | 危险区 UI 提前预警，结局页解释因果。 |
| 卡池太薄 | 前几局重复严重 | M4 前不要做公开试玩。 |
| 美术资源不够 | 角色复用过多削弱事件链 | 统一记录到 `MVP_Asset_Backlog.md`，集中生成。 |
| 系统变重 | 不再像 Reigns，而像经营游戏 | MVP 禁止物品栏、地图、战斗和多角色统治者。 |

## 下一次实现建议

下一次进入代码实现时，先做 M0 + M1：

- 用 seed JSON 跑通静态数据。
- 做一个可拖动事件卡。
- 做资源 HUD。
- 做选择应用和下一张卡。

只要 M1 成立，后面概率袋、结局和内容扩充都能顺着加。
