# 《朕的一生》MVP 设计对照报告 v0.1

日期：2026-04-20

本报告对照以下基线：

- 原始产品与策划文档：`PRD_Chinese_Reigns.md`、`GameDesign_Chinese_Reigns.md`
- Reigns 复刻约束：`Reigns_Reference_Analysis.md`
- MVP 范围与实施目标：`MVP_Implementation_Spec.md`、`MVP_Build_Plan.md`
- 当前实现：`public/index.html`、`public/main.js`、`public/engine.js`、`data/chinese-reigns/*.json`

目标不是判断“做得好不好”，而是回答两个更具体的问题：

1. 现在的版本有没有偏离最初设计？
2. 现在的版本是不是仍在尽量复刻《王权 Reigns》的成功体验？

## 一、结论先行

整体判断：**核心玩法已经明显对齐 Reigns，但产品包装、UI 引导、王朝延续感这三块还没有完全对齐最初策划。**

一句话概括当前状态：

> 它已经是一个“能玩出 Reigns 骨架”的版本，但还不是一个“从进入页面开始就让人立刻觉得这就是中国版 Reigns”的版本。

## 二、总评分

| 维度 | 当前判断 | 说明 |
| --- | --- | --- |
| Reigns 核心循环复刻度 | 高 | 一屏一卡、左右二选一、四资源高低双危险、频繁结局、快速重开，骨架已经成立。 |
| 产品范围控制 | 高 | 没有滑向菜单经营、地图、战斗、背包，守住了 MVP 边界。 |
| 数据驱动与扩展性 | 高 | 事件、结局、目标、规则都基本走 JSON 和 engine 规则。 |
| UI/UX 完成度 | 中上 | 移动端一屏、滑动、结果页都已成型，但开场仪式感和卡牌戏剧性还可更像 Reigns。 |
| 内容量与节奏 | 中上 | 48 张卡、12 个结局，已超过最初 40 卡目标，但真实游玩中的“故事网络感”还需继续调平。 |
| 王朝/世代感 | 中 | 有“下一任皇帝”和本地档案，但尚未真正形成强烈的 dynasty 传承感。 |
| 传播感 | 中 | 本地结果图已可生成，但还没完全达到原 PRD 里“朋友圈测试图”的传播完成度。 |

## 三、从产品经理角度看

### 3.1 已经对齐原始设计的地方

1. **MVP 范围控制是正确的。**
   当前实现保留了“一屏一卡、左右滑、四资源、不显示精确数值、快速结局”的主轴，没有加入地图、背包、战斗、经营面板，符合 `MVP_Implementation_Spec.md` 和 `Reigns_Reference_Analysis.md` 的锁定规则。

2. **核心成功指标的第一层已经具备。**
   从实现上看，用户进入后可以直接开始、滑动决策、看到资源变化、进入结局、再开新局；这已经满足“30 秒理解规则”和“完成一局”的基本门槛。

3. **产品形态已经不是普通经营游戏。**
   当前运行时把注意力集中在卡牌选择与后果上，而不是复杂系统管理，这一点非常关键，说明产品方向没有跑偏。

### 3.2 和最初设计相比的偏离点

1. **“世代循环”还偏弱。**
   原 PRD 和 GDD 里提到“下一任皇帝”“部分世界状态继承”“跨世代感”；当前实现里新一任主要是重开并保留档案记录，还没有形成强烈的 dynastic continuity。

2. **“结果传播”只完成了一半。**
   现在有结果页和本地结果图，但距离 PRD 中设想的“海报式传播体验”还有一点差距，尤其是更鲜明的人格化标签、王朝时间线感、可晒性文案层次。

3. **“中国化叙事网络”有雏形，但还不够密。**
   目前已经有宦官、太后、太子、方士、边关、灾荒等链路，但还没有完全形成文档里那种“朝廷、后宫、藩镇、民生、奇谈”彼此交错的密集故事网。

### 3.3 产品经理结论

从 PM 角度，当前实现**没有偏离项目目标**，反而是很健康地在“先像 Reigns，再慢慢长内容”这条路上。真正要警惕的不是“做歪了”，而是：

- 现在的产品已经能玩，但还不够“记住这个王朝”；
- 现在的产品已经有结局，但还不够“想拿去晒”；
- 现在的产品已经有事件链，但还不够“像一台会记仇的宫廷叙事机器”。

### 3.4 产品建议优先级

1. 强化 dynasty 感：让“下一任皇帝”不仅是重开，而是带有上一任遗产的轻微延续。
2. 强化结果页传播感：让称号、评语、死因、在位年数的组合更像人格测试结果。
3. 强化事件网络密度：优先补会彼此咬合的宫廷链，而不是继续均匀加散卡。

## 四、从 UI/UX 角度看

### 4.1 已经做对的地方

1. **主界面结构符合移动 H5。**
   当前游戏界面把 header、资源 HUD、目标、卡牌、左右按钮压进一屏，方向是对的。

2. **资源展示遵守了“只给方向，不给数字”。**
   HUD 使用图标和条形长度，不直接展示具体数值，这很好地保住了 Reigns 的直觉式决策感。

3. **卡牌交互是目前最像 Reigns 的部分。**
   左右拖动、阈值提交、拖动时显示资源方向预览，这一段已经很接近目标体验。

4. **视觉基线是统一的。**
   角色、背景、结局图已经基本锁定在 SBTI-lowpoly 宫廷人格测试风格里，整体不会再出现大幅跑偏。

### 4.2 还不够像 Reigns / 不够像最初设计的地方

1. **开场页更像 H5 首页，没那么像“马上进入统治”。**
   现在有标题、简介、登基按钮，这没有问题，但缺少第一张卡立刻把玩家卷进去的戏剧感。

2. **卡牌本体的信息层次还比较克制。**
   当前事件卡主要是背景、头像、标题、正文，已经够用，但“角色正在向你发问”的压迫感还不够强。

3. **左右选择的个性还可以更鲜明。**
   现在左右 hint 与按钮能工作，但离《王权》那种“拖一点点就感到两个答案性格不同”的体验还有空间。

4. **目标条有价值，但略微暴露“任务感”。**
   目前“本任目标”在可用性上没问题，但从纯 Reigns 味道来看，它稍微把界面往“任务系统”推了一点。后续更适合继续轻量化表达。

5. **结果页的信息排序还可以更狠一点。**
   现在结果页先出图、结局名、称号、总结、年数、解锁数，是标准可用方案；但如果要更像人格测试传播图，称号与一句话 punchline 的存在感还可以更强。

### 4.3 UI/UX 结论

从 UI/UX 角度，当前版本已经做到**可用、清晰、适配手机**；但如果我们目标不是“一个能玩的 H5”，而是“尽量复刻 Reigns 的感受”，那还需要继续推进：

- 更强的开场代入；
- 更强的卡牌戏剧性；
- 更强的结果页人格化呈现。

### 4.4 UI/UX 建议优先级

1. 把开场页进一步收紧，尽快让用户进入第一张卡。
2. 强化卡牌上角色发问的存在感，减少“静态海报”感。
3. 调整结果页层级，让称号和分享文案更像“可晒结果”。

## 五、从普通玩家角度看

### 5.1 玩家会立刻感受到的优点

1. **上手很快。**
   玩家几乎不用学习，就知道“左右选一个”。

2. **每次选择都有风险。**
   资源不显示具体数值，会逼玩家用直觉判断，这很像 Reigns 的乐趣来源。

3. **结局已经有“想再来一局看看别的死法”的基础。**
   12 个结局加上结果图，已经足够形成第一层复玩动机。

4. **美术会让人记得住。**
   这套 SBTI-lowpoly 的人格测试气质，确实和市面上常见古风 H5 区分开了。

### 5.2 玩家可能会觉得还差一口气的地方

1. **还不够“会记得我做过什么”。**
   玩家现在知道自己这一局结束了，但不一定强烈感到“这是一个王朝接着一个王朝留下的烂摊子”。

2. **部分事件链已经有了，但宿命感还不够浓。**
   真正优秀的 Reigns 体验会让玩家逐渐意识到“这事是我几年前埋下的雷”。我们现在有这个方向，但还可以更明显。

3. **结果页还不够“我想发给朋友”。**
   玩家会看结果，但未必立刻觉得“这图很像一个人格测试结果，值得转发”。

4. **现在的版本更像“不错的测试版”，还不是“黏人的正式版”。**
   核心体验对了，但长期记忆点还需要更尖一点。

### 5.3 玩家结论

普通玩家大概率会觉得：

> 这个游戏好懂、会想多玩几把，而且确实像《王权》的中国宫廷版。

但他们也可能同时觉得：

> 现在已经好玩了，不过还没到“我会和朋友讨论我上次是怎么死的”那个强度。

## 六、当前实现与原始设计的对照

### 6.1 对齐度高的项目

- 一屏一卡，左右二选一。
- 每张卡推进一年。
- 四资源过高过低都能触发结局。
- HUD 不显示精确数值。
- 事件按条件、权重、once、cooldown 抽取。
- 有事件链与隐藏状态。
- 死亡后能快速再开一局。
- 有结局解锁、本地记录、结果页、结果图。
- 移动端一屏体验已做过回归。

### 6.2 部分对齐、但还需继续补强的项目

- 本任目标系统已实现，但表达还可更轻。
- 稀有善终已实现，但仍需继续看真实玩家手感。
- 宫廷链路已经有雏形，但整体故事网络密度还可以再加强。
- 分享图已能生成，但传播气质仍有提升空间。

### 6.3 明确还未完全对齐的项目

- 跨世代继承感仍弱，尚未真正形成“dynasty meta”。
- 开场的第一分钟体验还不够像 Reigns 那样干脆、锐利。
- 玩家对“因果报应”的感知还没有完全拉满。

## 七、是否仍在尽量复刻《王权》？

答案是：**是，而且方向基本正确。**

原因有三点：

1. 我们目前做出的东西，本质上仍然是 **Reigns 核心公式的中国题材变体**，而不是借题发挥去做另一种策略游戏。
2. 绝大多数新增内容，比如太后线、太子线、方士线、资源高值结局，都是在强化 Reigns 的“短决策 + 长后果”。
3. 当前真正的不足，不是“偏离 Reigns”，而是“还没有把 Reigns 那种宿命感、王朝感、传播感 fully 打出来”。

所以更准确的说法不是“我们做偏了”，而是：

> 我们已经把 Reigns 的骨架搭起来了，现在该做的是把它从“对的系统”打磨成“对的感受”。

## 八、下一步建议

### P0：最优先

1. 做一轮真人 playtest，记录玩家是否真正感受到：
   - 左右选择都痛；
   - 资源不能贪高；
   - 某些结局是自己早年埋下的雷。
2. 继续调结果页层级，让称号和一句话点评更可晒。
3. 增加一层轻量 dynasty 表达，让新皇登基时更像承接前朝，而不是普通重开。

### P1：紧接着做

1. 强化 2-3 条最关键事件链的“因果可见度”。
2. 让首张卡和前 3 张卡的引导节奏更戏剧化。
3. 继续压缩任何会让界面显得像“任务清单”的表达。

### P2：后续优化

1. 继续补充高质量的宫廷散卡与桥接卡。
2. 进一步提高分享图的社交传播完成度。
3. 在不破坏 Reigns 轻量感的前提下，增加一点跨代遗产痕迹。

## 九、最终判断

如果把最初目标拆成两层：

- **系统层**：复刻 Reigns 的玩法公式；
- **感受层**：复刻 Reigns 的宿命感、荒诞感、再来一局冲动；

那么当前版本的状态可以总结为：

- **系统层：已经大体达标。**
- **感受层：已经有七成到八成轮廓，但还差最后一段打磨。**

这不是坏消息，反而说明项目现在最难的方向判断已经做对了。接下来我们要做的，不是推翻重来，而是继续把“像”这件事做得更彻底。
