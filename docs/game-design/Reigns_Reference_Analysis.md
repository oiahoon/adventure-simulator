# Reigns 参考分析与《朕的一生》复刻原则 v0.1

日期：2026-04-20

本文基于 Reigns 原作、后续版本和玩家/媒体分析整理，用于约束《朕的一生》MVP：第一版要尽可能复刻 Reigns 的成功玩法骨架，再做中国皇权题材换皮与轻量创新。

## 1. 参考来源

- Reigns App Store / Google Play：原作的滑动统治、四势力平衡、每年一张请求、决策后果与王朝风险。
- Reigns: Her Majesty Steam / App Store：王后视角、物品栏、每任君主目标、解锁新卡组。
- Reigns: Game of Thrones Steam：多位统治者、IP 角色专属剧情、小游戏和更大的剧情体量。
- Reigns: Three Kingdoms Steam / Nintendo / Netflix：三国题材、军事谈判、联姻、招募、子嗣、回合制卡牌战斗和小游戏。
- Reigns: Beyond Nintendo：同样的滑动公式扩展到太空乐队，强调 60+ 角色、1400+ 卡、资源与船员管理。
- GamesRadar、Vice、Gamespresso、App Store 评价等玩家/媒体分析：二元操作很简单，但真正吸引力来自隐藏因果、荒诞死亡、跨代进度和“看似随机但有反应”的事件网络。

## 2. Reigns 原作必须复刻的核心

### 2.1 两输入统治

每张卡只有两种处理方向。玩家的主要动作就是左右滑动，不进入菜单、不读长说明、不做复杂配置。

对《朕的一生》的要求：

- 主循环只允许左右滑。
- 左右选项文案短，像“准奏/驳回”“开仓/观望”。
- 不在主循环中加入多按钮、背包、技能树、地图等额外操作。

### 2.2 四条资源的中庸压力

Reigns 的关键不是把资源堆高，而是四条资源过低或过高都危险。玩家永远在中间区域摇摆。

对《朕的一生》的要求：

- 民心、国库、兵权、朝政都必须有低值结局和高值结局。
- UI 不显示精确数字，优先显示图标条和危险感。
- 每个选择只预告影响方向，不显示具体数值。
- 玩家不能通过“永远选加资源”轻松胜利。

### 2.3 每回合像一年

Reigns 把每张请求包装成统治期间的一年或一段时间，让极短的选择自然积累成一生。

对《朕的一生》的要求：

- MVP 中每处理一张卡，`year += 1`。
- 结果页强调“在位 X 年”。
- 卡牌文案要像朝堂纪事，而不是普通问答。

### 2.4 死亡不是失败，而是内容推进

Reigns 的死亡很频繁，但死亡会解锁记忆、目标和新路径。玩家不是为了不死，而是为了探索更奇怪的死法和更长的统治。

对《朕的一生》的要求：

- 结局页必须有死因、称号和一句可分享点评。
- 本地记录已解锁结局。
- 下一任皇帝立即开始，降低死亡挫败感。
- 死亡原因要荒诞、具体、有传播欲。

### 2.5 看似随机，其实被状态牵引

Reigns 的卡牌呈现像随机洗牌，但玩家状态、过往选择、角色关系和事件链会改变后续卡池。

对《朕的一生》的要求：

- MVP 必须实现条件过滤、权重、once、cooldown。
- 危险资源提高相关危机卡和补救卡权重。
- 事件链必须有前因后果，不能只是随机散卡。
- 前 10 张 seed 卡用于验证系统，40 张 MVP 卡用于验证重复度。

### 2.6 幽默和痛感并存

Reigns 的选择经常荒诞，但后果真实。玩家被迫在两个都不完美的答案里选一个。

对《朕的一生》的要求：

- 事件文本短，但必须有角色口吻和笑点。
- 左右选项都要有代价。
- 不写“正确答案”和“错误答案”。
- 死因可以黑色幽默，但不要恶俗。

## 3. 各版本可借鉴但不应抢 MVP 的扩展

### 3.1 Her Majesty：目标与物品栏

可借鉴：

- 每任君主开始时给 2-3 个目标。
- 完成目标解锁新卡或新结局。

MVP 决策：

- 采用“本任目标”。
- 不做物品栏。物品栏会破坏第一版的极简滑动节奏。

### 3.2 Game of Thrones：多统治者与专属剧情

可借鉴：

- 不同统治者有不同剧情和死亡方式。
- 角色专属分支能降低重复感。

MVP 决策：

- 暂不做多皇帝人格/身份系统。
- 只在结果页称号中体现玩家风格。
- 后续可扩展“开局身份”或“皇帝特质”。

### 3.3 Three Kingdoms：三国题材扩展

可借鉴：

- 中国历史题材可以自然接入联姻、招募、战斗、子嗣。
- 任务 arc 和英雄关系能形成长期目标。

MVP 决策：

- 暂不做回合制卡牌战斗。
- 暂不做招募英雄系统。
- 联姻/立储只作为事件链和 hidden state 存在。

### 3.4 Beyond：大内容量和变体场景

可借鉴：

- 大量角色和卡牌是长期留存的基础。
- 公式可以换题材，但必须保留资源管理和死亡循环。

MVP 决策：

- 第一版先做到 40 张卡可玩。
- 内容规模不足时不要公开传播。

## 4. 《朕的一生》MVP 复刻原则

### 4.1 必须像 Reigns

- 一屏一张卡。
- 只有左右滑。
- 四条资源都可过低/过高致死。
- 每张卡推进一年。
- 死后下一任皇帝继续。
- 结局收集和称号驱动复玩。
- 当前选择只显示方向，不显示数值。
- 随机抽卡由状态牵引。

### 4.2 必须不像普通经营游戏

- 不做建筑菜单。
- 不做数值详情页。
- 不做复杂养成。
- 不做战斗系统作为 MVP 核心。
- 不把事件做成长剧情弹窗。

### 4.3 中国化方式

资源映射：

- Church -> 朝政/士林/礼法。
- People -> 民心。
- Army -> 兵权。
- Treasury -> 国库。

主题映射：

- Advisors -> 宰相、老臣、言官、宦官、将军、贵妃、百姓。
- Dynasty -> 改朝换帝/下一任皇帝。
- Deaths -> 民变、政变、逼宫、国库崩溃、服丹暴毙、边关失守。
- Royal Challenges -> 本任年号目标/祖宗遗训/天命任务。

## 5. MVP 更新要求

基于以上分析，MVP 文档需要增加：

- Reigns 复刻约束。
- 本任目标系统。
- 死亡/退位后的时间线记录。
- 高值结局与低值结局成对设计。
- 不显示具体数值的 UI 规则。
- 至少 40 张卡后再做公开试玩。

## 6. 后续资料复查点

- 玩家对 Reigns: Three Kingdoms 的 mixed 评价提示：三国战斗和系统扩展可能让公式变重。我们应先复刻原版轻量循环，再考虑战斗。
- 玩家对 Reigns 原作和 Her Majesty 的正面评价集中在“简单、聪明、幽默、总想再来一局”。MVP 的所有功能都要服务这个体验。
